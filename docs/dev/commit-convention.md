# Commit Convention

Use concise semantic commits with explicit task or area scope.

Examples:

- `feat(task1-js): add four-marker field pose estimation`
- `feat(debug-js): add marker axis gizmos and field grid`
- `feat(task4-js): add ray-plane shadow projection`
- `feat(task2-py): add mock object detection endpoint`
- `docs(debug): describe developer mode overlays`
- `refactor(shared-math): extract plane intersection utilities`

Rules:

- prefer `feat`, `fix`, `docs`, `refactor`, `test`, `chore`;
- include task scope when it clarifies ownership;
- keep one logical change per commit.
